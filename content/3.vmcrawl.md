## Crawled Endpoints

vmcrawl may attempt to access the following endpoints of your Fediverse server:

- `/robots.txt`
- `/.well-known/webfinger`
- `/.well-known/nodeinfo` (and the discovered `/nodeinfo/2.x`)

vmcrawl may periodically attempt to access the following endpoint from known deployments:

- `/api/v1/instance/peers`

## User Agent

The vmcrawl user agent can be identified as: `vmcrawl/0.x (https://docs.vmst.io/vmcrawl)`

### robots.txt

If you would like to stop vmcrawl from checking your server, you can add the following to your `robots.txt` file:

```
User-agent: vmcrawl
Disallow: /
```

It will also respect a disallow of all bots/crawlers in this file, or an HTTP 410 or 418 reply to any endpoint.

## Collected Data

vmcrawl collects the following data only from servers that it identifies as Mastodon, or a related fork:

- Software Version
- Total Users Count
- Monthly Active Users Count

vmcrawl may periodically request a list of peers from a server, but only to discover new servers to request the data outlined above.

**vmcrawl does not store specific peer information for any server.**

## Uncrawled Servers

Some servers may not appear in our results based on many factors:

- Servers may opt-out by blocking our user agent as outlined above
- Servers may be on sub-domains which are known to be development instances
- Servers may be running older versions of Mastodon which lack the proper APIs
- Servers may report distorted version or user count information
- Servers may have an improperly configured WebFinger configuration (see `WEB_DOMAIN` in [Mastodon Documentation](https://docs.joinmastodon.org/admin/config/#web_domain))

## Our Instance

vmcrawl is available for anyone to run on their own server.
If you have concerns about our particular implementation, and would like to permanently opt-out in a way not outlined above, please contact us at [admin@vmst.io](mailto:admin@vmst.io).

vmcrawl runs on the same backend [infrastructure](/about/infrastructure) as the vmst.io Mastodon instance, in DigitalOcean's Toronto data center.
There are also virtual collectors running in DigitalOcean's Sydney and Amsterdam data centers and in Hetzner's Oregon, data center.
